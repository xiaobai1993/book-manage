## 📝 文档分类

### PRD（产品需求文档）
- 描述功能需求、业务规则、用户体验等
- 文件命名：`{功能名称}PRD.md`

### API 文档
- 描述接口规范、请求参数、响应格式等
- 文件命名：`API_{功能模块}.md`

### 实施方案
- 描述技术实现方案、配置步骤等
- 文件命名：`{技术方案名称}_实施方案.md`

### 操作指南
- 描述详细的操作步骤、配置方法等
- 文件命名：`{功能名称}_操作指南.md` 或 `{平台名称}配置指南.md`

### 测试文档
- 描述测试方法、测试步骤等
- 文件命名：`{功能名称}_测试指南.md`

### 迁移文档
- 描述数据库迁移、数据迁移等
- 文件命名：`数据库迁移指南_{功能名称}.md`

## 🆕 新增需求文档

当需要新增功能需求时，请按以下步骤：

1. **创建需求文件夹**
   ```bash
   mkdir -p requirements/{功能名称}
   ```

2. **创建 PRD 文档**
   - 文件：`requirements/{功能名称}/{功能名称}PRD.md`
   - 包含：功能概述、需求描述、业务规则等

3. **创建 API 文档**（如需要）
   - 文件：`requirements/{功能名称}/API_{功能模块}.md`
   - 包含：接口规范、请求/响应格式等

4. **创建实施方案**（如需要）
   - 文件：`requirements/{功能名称}/{技术方案}_实施方案.md`
   - 包含：技术方案、配置步骤、代码实现等

5. **创建其他相关文档**
   - 测试指南、迁移指南、操作指南等
