# Render 手动部署配置指南

## 🎯 关闭自动部署

Render 默认会在你推送代码到 GitHub 时自动部署。如果你想改为手动部署，可以按以下步骤操作：

### 步骤 1：进入服务设置

1. 登录 [Render Dashboard](https://dashboard.render.com)
2. 点击你的服务（例如 `book-manage-backend`）
3. 点击顶部的 **Settings** 标签

### 步骤 2：关闭自动部署

在 Settings 页面中，找到 **Auto-Deploy** 部分：

1. 找到 **Auto-Deploy** 开关
2. 将开关切换到 **关闭**（Off）状态
3. 点击 **Save Changes** 保存

### 步骤 3：手动部署

关闭自动部署后，当你需要部署新代码时：

1. 在服务页面，点击 **Manual Deploy** 按钮
2. 选择部署选项：
   - **Deploy latest commit** - 部署最新的提交
   - **Deploy specific commit** - 部署指定的提交（输入 commit hash）
3. 点击 **Deploy** 开始部署

---

## 📋 自动部署 vs 手动部署

### 自动部署（Auto-Deploy）
- ✅ 推送代码到 GitHub 后自动部署
- ✅ 适合快速迭代开发
- ❌ 可能在不合适的时间触发部署
- ❌ 无法控制部署时机

### 手动部署（Manual Deploy）
- ✅ 完全控制部署时机
- ✅ 可以在测试通过后再部署
- ✅ 避免意外部署
- ❌ 需要手动操作

---

## 🔄 重新启用自动部署

如果以后想重新启用自动部署：

1. 进入服务 Settings
2. 找到 **Auto-Deploy** 部分
3. 将开关切换到 **开启**（On）状态
4. 保存更改

---

## 💡 推荐配置

对于生产环境，建议：
- **生产环境**：使用手动部署，确保稳定性
- **开发/测试环境**：可以使用自动部署，快速迭代

---

## 📝 注意事项

1. 关闭自动部署后，每次代码更新都需要手动触发部署
2. 手动部署时，Render 会拉取你指定的 commit
3. 可以随时切换回自动部署模式
